# 📄 Annex E — Sandbox Engagement Protocol

## 🧭 Purpose

This annex defines how external laboratories, research institutions, and sandbox environments may interact with the Sextant Orbital Resilience Framework.

It establishes **clear separation between simulation logic and external execution environments**.

---

## 🧠 Core Principle

All interactions with the framework must occur under:

> Controlled, non-operational, sandboxed simulation conditions only.

The framework does not interface with or control live systems.

---

## 🧪 1. Engagement Model

External parties interact with the framework through:

- predefined input structures (Annex A)
- dependency logic definitions (Annex B)
- cascade rules (Annex C)
- output interpretation models (Annex D)

No direct modification of core logic is required.

---

## 📦 2. Role Separation

### Framework Owner (Sextant System)
- defines simulation structure  
- provides deterministic logic  
- maintains model integrity  

### External Lab / Agency
- executes simulation in controlled environment  
- provides input datasets  
- evaluates outputs  
- returns feedback on system behaviour  

---

## 🧱 3. Execution Boundary Rule

The framework must NOT be used for:

- live system control  
- operational decision-making  
- real-world infrastructure manipulation  
- real-time mission execution  

All execution is strictly simulation-only.

---

## 🔁 4. Sandbox Execution Flow

Typical engagement flow:

1. Lab defines input scenario (Annex A format)  
2. Framework processes simulation logic  
3. Cascade propagation executes (Annex C)  
4. Outputs are generated (Annex D)  
5. Lab evaluates system behaviour  
6. Feedback is returned for model refinement  

---

## 🧠 5. Deterministic Constraint Requirement

All sandbox executions must ensure:

- identical inputs → identical outputs  
- no stochastic variation  
- no external environmental dependency  
- reproducibility across test environments  

---

## 🧩 6. Interface Limitation Principle

External systems:

- may not alter internal simulation logic  
- may not inject runtime behavioural changes  
- may only interact via defined input/output schemas  

---

## 📊 7. Evaluation Scope

External evaluation may include:

- cascade behaviour validation  
- dependency robustness testing  
- resilience scoring comparison  
- scenario stress testing  

---

## 📌 8. Governance Boundary Statement

This framework is a **simulation and modelling system only**.

It is designed for:

- research validation  
- academic modelling  
- controlled sandbox experimentation  

It is not an operational control system.

---

## 🧭 Summary

This annex defines:

- how external labs interact with the system  
- what boundaries must not be crossed  
- how sandbox execution is structured  
- how deterministic reproducibility is preserved
